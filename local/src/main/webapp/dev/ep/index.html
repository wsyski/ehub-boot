Background

Axiell eHUB is a media proxy between media providers and end users. It needs to call the connected media providers through their API thus connecting eHUB to their providers. The current scenario is that the media provider has already its API (which fulfills the eHUB requirements) and the adapter code is written by Axiell Developers and added to eHUB. This is usually time consuming both in terms of writing a new code and communicating with the Provider Developers. Also a new version of eHUB needs to be deployed each time a new provider is implemented.

The above scenario disadvantages forced a new approach, where a new eHUB Provider API (EP API) interface is designed. The main idea is that if the media provider delivers a server code implementing EP API then no coding is required at the eHUB site and the new media provider could be added just by submitting the EP parameters from eHUB Administration Pages. No new version of eHUB needs to be deployed, and the media provider data is persisted in the eHUB database.
