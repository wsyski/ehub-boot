eHub Formats API

The Formats API delivers a list of the formats for a specific record.

Use the query parameters detailed below to decide in which language the names and the descriptions should be translated in. If no language is provided the formats will be delivered in a default language.

Accessing the Formats API

All you need to start is the name of the Content Provider as defined by the eHub and the ID of the record at the Content Provider that you want to get the available formats for.

Resource URL

https://eHub.axiell.com/v1/content-providers/{Content Provider Name}/records/{Content Provider Record ID}/formats?parameters

Query Parameters

language The ISO 639 alpha-2 or alpha-3 language code used when getting the translated names and descriptions of the formats

How to return a list of available formats

The Formats API can be utilized to return a list of formats for a record by accessing the formats endpoint.

GET https://eHub.axiell.com/v1/content-providers/atingo/records/123456/formats?language=en
Authorization: eHub eHub_consumer_id="1", eHub_library_card="909265910", eHub_pin="4447", eHub_signature="GN%2B9mlD70ZER%2Fx3ur7w7HJRgnYU%3D"

A call to the formats endpoint returns a list of formats as shown in the example below.

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<eHub:contentProviderFormats xmlns:eHub="http://eHub.axiell.com">
    <contentProviderFormat description="description1" iconUrl="iconUrl1" id="id1" name="name1"/>
    <contentProviderFormat description="description2" id="id2" name="name2"/>
</eHub:contentProviderFormats>
