  4. eHUB Provider API Parameters
  • The following parameters are used for each eHUB Provider API instance:
    API_BASE_URL eHUB Provider API base url ex. if API_BASE_URL="https://provider.com/ehub-api", then the checkout url is: "https://provider.com/ehub-api/v1/checkouts".
  • The following parameters are used for each eHUB Provider API instance and eHUB Consumer combination:
    EP_SITE_ID Unique site identifier at the eHUB Provider. If the provider does not use this parameter then it can be set to any nonempty value. This value is always sent in the Authorization Http header in every request originating at eHUB. See the security page for more information.
    EP_SECRET_KEY A UTF-8 encoded secret key used to produce the signature in the Authorization Http header in every request originating at eHUB. The string is converted into bytes and then used to produce HMAC-SHA1 digest of the Authorization http header. See the security page for more information.
    EP_TOKEN_EXPIRATION_TIME_IN_SECONDS The valid values of this parameter are: LIBRARY_CARD and PATRON_ID. In the first case the patron library card (bar code) is used as the user id. In the second case the Local Management System (LMS) generated id is used as user id. See the discusion in eHUB Provider API Users page for more information.
